# Screenshot assets

This folder can temporarily contain **public-safe placeholder screenshots** during the open-source prep phase.

For the first experimental public release, the expected filenames are:

- `login-screen.png`
- `desktop-home.png`
- `desktop-edit-bookmark.png`
- `mobile-home.png`

Right now, these files may be one of two things:

1. real screenshots captured from the product
2. temporary stand-in assets with the final filenames already locked in

That lets README/docs embed stable paths now, while still allowing higher-quality real screenshots to replace them later without churn.

See `docs/product/first-public-release.md` for:

- the concrete capture checklist
- the operator-ready capture runbook
- screenshot content expectations
- related demo/GIF plans
